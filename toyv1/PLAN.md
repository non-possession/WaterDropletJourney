# `toyv1` 水滴单体动作与灌木互动实现计划

## Summary

在仓库根目录新增 `toyv1/PLAN.md`，内容为“水滴单体动作 + 灌木互动小演出”的实现方案；随后按该方案实现一个**独立测试场景**，验证：

- 水滴单体在 2D 场景中的核心六态动作
- 水滴与灌木的“停驻-回应”互动
- 镜头、粒子、音频如何共同支撑这种低压、诗性的关系体验

本轮执行目标不是接入完整第一章，而是交付一个**可独立运行、可复用、可嵌回第一章**的互动切片。

## Key Changes

### 1. 新增 `toyv1/PLAN.md`

文档内容直接采用当前已经确定的方案，不再留给实现者做设计判断。  
文档应包含：

- 独立测试场景目标
- 水滴核心六态
- 灌木互动逻辑
- 镜头与音频规则
- 可复用接口
- 验收标准

### 2. 新建独立测试场景

新增一个专用 toy 场景，不修改第一章主流程。  
建议结构：

- `ToyV1Root`
- `Camera2D`
- `Environment`
- `Player`
- `BushInteraction`
- `FX`
- `Audio`

场景只保留最小可玩空间：

- 一小段地表
- 一簇可互动灌木
- 轻背景和少量前景
- 玩家出生点与离开空间

### 3. 扩展 `WaterPlayer` 的表现层

在现有 `WaterPlayer` 上新增轻量表现状态接口，不重写成大状态机。  
必须支持：

- `idle`
- `move`
- `pause`
- `approach`
- `nestle`
- `leave`

实现方式：

- 保留现有 `AnimatedSprite2D`
- 用 Tween / scale / offset / glow / 粒子变化实现状态差异
- 输入与表现解耦：输入只负责移动意图，互动对象可以临时覆盖表现状态

对外接口固定为：

- `set_expression_state(state_name: String)`
- `clear_expression_state()`

### 4. 实现 `BushInteraction` 可复用场景

新增一个独立灌木互动对象，做成 `PackedScene`，而不是写死在 toy 场景里。  
第一版逻辑固定为：

- 水滴进入探测范围：灌木进入轻呼吸
- 水滴停驻达到阈值：灌木回应完成
- 回应完成时：
  - 灌木亮度上升
  - 叶片轻微摇曳
  - 水滴切入 `nestle`
  - 附近粒子更贴叶面
  - 响起一层极轻的近距回应声
- 水滴离开时：
  - 灌木亮度回落
  - 摇曳停止
  - 水滴切入 `leave` 再回 `move/idle`

导出参数固定为：

- `detection_radius`
- `settle_time`
- `glow_strength`
- `sway_amount`
- `auto_reset_delay`

信号固定为：

- `interaction_started`
- `interaction_settled`
- `interaction_ended`

### 5. 加入最小镜头与音频逻辑

镜头规则：

- 默认轻跟随玩家
- 靠近灌木时减慢跟随
- 进入 `nestle` 时执行一次轻微 push-in
- 离开时缓慢恢复

音频规则：

- 复用现有主角待机/移动水声
- 为灌木加一条轻回应层
- 不做机关式提示音
- 保持近、柔、低量

### 6. 资源使用策略

优先复用当前已有资源，不新开大规模美术需求。  
默认资源策略：

- 主角：复用当前 `water_player` / `water_states`
- 灌木：从 `新灵气版本` 中提炼风格参考，但本轮实现使用可分层的独立 2D 资源，而不是直接把整张插画塞进场景
- 粒子：沿用当前水滴粒子语言，增加贴近草叶的小范围发光点
- 背景：保持轻，不抢互动主体

如果灌木缺少可分层素材，实现时先用占位分层资源，但结构必须按最终可替换方式搭好。

## Test Plan

### Core Behavior

- 玩家进入 toy 场景后能自由控制水滴左右移动
- 水滴静止时有呼吸感，移动时有轻流动感
- 靠近灌木时能明显进入 `approach`
- 停驻达到阈值后能稳定进入 `nestle`
- 离开灌木时能稳定进入 `leave` 并恢复正常

### Interaction Feel

- 灌木回应不是跳变，而是渐入
- 玩家不看说明，也能理解“这里可以停留”
- 互动不是机关触发感，而是关系成立感
- 停驻后画面比移动时更安静、更贴近，而不是更花

### Reuse / Regression

- `WaterPlayer` 原有第一章基础移动不被破坏
- `BushInteraction` 可拖入其他场景复用
- 不依赖 `main.gd` 的第一章镜头逻辑
- 第一章现有资源、音频、场景不被强耦合修改

## Assumptions

- 先做独立 toy 场景，不接入第一章
- 灌木互动只做“停驻-回应”，不做“渗透-吸收”
- 水滴动作只做核心六态，不扩展完整情绪库
- 优先使用现有资源与轻量程序表现，不引入复杂 shader 方案作为前置条件
- 当前处于 Plan Mode，这一轮只交付决策完成的实施计划；退出 Plan Mode 后再创建 `toyv1/PLAN.md` 并执行实现
